# おうちクラウドサービス on Proxmox VE 9

## 概要
* マルチテナント
* 管理画面
    * DBには必要最低限の情報以外保存しない
        * 基本的にはProxmoxにある情報がベース
* 機能
    * 管理画面
    * VM管理
        * Cloud-initによるデプロイなど
    * DBaaS
        * MySQL / Postgres / Redis etc..（その他は今後拡大）
        * バージョンアップデート機能
        * 管理用S3に定期的にバックアップ（sqldumpなど）
            * 出来れば暗号化したい
    * コンテナ基盤（できればNomadベース、厳しそうならk8sベース。マルチテナント）
        * コンテナレジストリ
            * 全テナントがアクセス可能
        * テナントのネットワーク上に接続
    * 監視
        * Grafana Cloudに送信するようにOTELで収集
        * ログ基盤
            * ストレージ圧縮のためS3にアーカイブできるようになど
    * VPS経由でのグローバルアクセス

### 技術仕様
* 管理画面
    * Laravel
    * JS/TSフレームワークはなるべく不使用の方向で
    * Proxmoxへのアクセスはライブラリ使わずに自分で Lib/Proxmox みたいな形でライブラリを実装
    * UIは TailwindCSSを利用
* スニペット保存サーバ
    * ProxmoxのAPIではCloud-init用のスニペットファイルの保存が不可のため、Proxmoxサーバに直接起動させるサーバ
    * Python製でよい

## 現状
* Proxmox VE 9 をインストールしたサーバが3台
* ルータ - L2スイッチ - Proxmox VE の構成
* VM用のネットワークとして 172.26.27.0/24 を用意/使用
* 外部からのHTTPアクセス / ゲームサーバへのTCP・UDPはいったん外部のVPSを経由してVPSにWireGurdでVPNをはっているproxy VM経由で各VMにアクセス


### ProxmoxVEの設定
* SDN
    * Zone
        * 種別：EVPN
* Ceph
    * ３台でクラスタ構成済み

### ネットワーク設定
* 172.26.27.0/24: VM用ネットワーク（グローバルへのアクセスも可）
* 172.26.26.0/24: 管理用ネットワーク（グローバルアクセスは不可）
